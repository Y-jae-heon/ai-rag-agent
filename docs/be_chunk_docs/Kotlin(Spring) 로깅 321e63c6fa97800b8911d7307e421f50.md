## Rule

- 코틀린에서는 `companion object`에 logger를 선언하거나, `kotlin-logging` 라이브러리를 사용합니다.
- Controller, Repository 계층에서는 굳이 로깅하지 않습니다. 필요하다면 P6Spy, AoP를 사용합니다.

```kotlin
// 방법 1: companion object (SLF4J 직접 사용)
class OrderService {
    companion object {
        private val log = LoggerFactory.getLogger(OrderService::class.java)
    }
}

// 방법 2: kotlin-logging (권장)
import mu.KotlinLogging

private val logger = KotlinLogging.logger {}

class OrderService {
    fun createOrder() {
        logger.info { "[Order] 주문 생성 완료 orderId=${order.id}" }
    }
}
```

## 레벨

**ERROR** — 즉시 대응이 필요한 장애 상황에서 사용합니다.

외부 시스템 연동 실패, 데이터 정합성 깨짐, 예상하지 못한 예외 등이 해당됩니다.

```kotlin
try {
    paymentClient.request(orderId)
} catch (e: PaymentException) {
    logger.error(e) { "[Payment] 결제 요청 실패 orderId=$orderId" }
    throw e
}
```

**WARN** — 당장 장애는 아니지만 주의가 필요한 상황에서 사용합니다.

비즈니스 예외(`CustomException`), 재시도 발생, deprecated API 호출 등이 해당됩니다.

```kotlin
logger.warn { "[Order] 재고 부족으로 주문 실패 productId=$productId, requested=$requestedQty, available=$availableQty" }
```

> **ERROR**, **WARN**에서 예외 객체는 반드시 첫 번째 인자로 넣어 스택트레이스가 출력되도록 합니다.
kotlin-logging에서는 `logger.error(e) { "message" }` 형태를 사용합니다.
> 

**INFO** — 비즈니스 흐름의 주요 분기점에서 사용합니다.

주문 생성, 결제 완료, 회원 가입 같은 상태 변경 이벤트가 해당됩니다. 단, 반복 호출되는 조회 API에서는 사용하지 않습니다.

```kotlin
logger.info { "[Order] 주문 생성 완료 orderId=${order.id}, memberId=$memberId, totalAmount=$totalAmount" }
```

**DEBUG** — 개발/디버깅 시 필요한 상세 정보에서 사용합니다.

쿼리 파라미터, 중간 연산 결과, 외부 API 요청/응답 등이 해당됩니다. 운영 환경에서는 기본적으로 꺼둡니다.

```kotlin
logger.debug { "[Order] 할인 계산 결과 originalPrice=$originalPrice, discountRate=$discountRate, finalPrice=$finalPrice" }
```
