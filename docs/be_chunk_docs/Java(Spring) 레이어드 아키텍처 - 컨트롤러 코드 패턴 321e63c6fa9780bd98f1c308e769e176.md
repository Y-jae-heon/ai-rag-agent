## Title

Java(Spring) 레이어드 아키텍처 - 컨트롤러 코드 패턴

## Rule

도메인별로 크게 `Controller` `Service` `Repository` 레이어로 역할이 나눠지며 상위 레이어에서 하위 레이어로만 의존방향이 향합니다.

## Controller

- 역할 : HTTP 요청 처리, 입력값 검증, 서비스 호출
- Swagger 문서화를 위해 `@Tags` `@Operation` 어노테이션을 활용합니다.
- 검증 : `spring-boot-starter-validation` 을 사용해 검증 어노테이션을 사용합니다.
- 경로 : 기본적으로 케밥 케이스를 사용하며, 자원은 복수형으로 표기합니다. 리소스 이름은 카멜케이스를 사용합니다.
    - `/sensors/{sensorId}/details`
- 리턴타입 : 일관성 있는 리턴타입을 위해 `ObjectResponse`, `ListResponse` 를 사용합니다.

**ObjectResponse.java**

```java
@Getter
@AllArgsConstructor
public class ObjectResponse<T> {
  T row;
}
```

**ListResponse.java**

```java
@Getter
@AllArgsConstructor
public class ListResponse<T> {
  List<T> rows;
  int count;
}
```

```java
@RestController
@RequestMapping("sensors")
@RequiredArgsConstructor
@SecurityRequirement(name = "bearerAuth")
public class SensorController {
  private final SensorService sensorService;

  @GetMapping("/{id}/details")
  @Operation(
      summary = "센서 상세 데이터 조회",
      description = "센서의 상세 데이터를 조회합니다. ISO String 사용"
  )
  public ResponseEntity<ObjectResponse<FindSensorDetailRes>> findSensor(
      @PathVariable("id") Long sensorId,
      @ModelAttribute FindSensorDetailReq dto
  ) {
    Sensor sensor = sensorService.findDetailById(sensorId, dto);    

    return ResponseEntity.ok(new ObjectResponse<>(FindSensorDetailRes.from(sensor)));
  }
}
```

**DTO**

- Query Parameter, Path Variable, Body 모두 DTO를 사용해서 받습니다.
- **Query Parameter** : 컨트롤러에서 `@ModelAttribute` 를 사용합니다. DTO에서 `@ParameterObject` 를 사용합니다.
- **Path Variable** : `@PathVariable` 을 사용합니다.
- **Body** : `@RequestBody` 를 사용합니다.

### 응답 코드

- 200 : 일반적인 데이터 리턴
- 201 : `POST` 요청으로 데이터가 생성되었을때
- 204 : 삭제, 수정등 작업 수행
- 400 : 잘못된 요청
- 401 : 인증 오류
- 403 : 인가 오류
- 404 : 자원 없음 (단일 자원일때)
- 500 : 예상하지 못한 서버 오류
