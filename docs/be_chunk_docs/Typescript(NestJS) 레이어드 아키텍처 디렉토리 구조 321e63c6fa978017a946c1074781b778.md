## Title

NestJS 레이어드 아키텍처 디렉토리 구조

## Rule

```
src/
├── config/              # 설정 파일 (환경 변수 등)
├── database/            # 데이터베이스 관련 (마이그레이션, 시드, 설정)
├── domain/              # 기능 도메인 (비즈니스 로직)
│   ├── [domain-name]/
│   │   ├── controllers/ # 요청 핸들러
│   │   ├── services/    # 비즈니스 로직
│   │   ├── repositories/# 데이터 접근 계층
│   │   ├── entities/    # 데이터베이스 엔티티
│   │   ├── dtos/        # 데이터 전송 객체 (DTO)
│   │   └── [domain].module.ts
├── shared/              # 공통 유틸리티, 상수, 데코레이터
│   ├── constants/
│   ├── decorators/
│   ├── dtos/            # 공통 DTO (응답 래퍼 등)
│   ├── filters/         # 예외 필터
│   ├── interceptors/
│   └── utils/
├── main.ts              # 애플리케이션 진입점
└── app.module.ts        # 루트 모듈
```

## Rationale

도메인별로 관련된 코드를 모아두기 때문에 응집도가 향상되고, 개발 및 유지보수성이 향상됩니다.
