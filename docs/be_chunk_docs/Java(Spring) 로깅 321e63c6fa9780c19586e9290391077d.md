## Title

Java(Spring) 로깅

## Rule

- `@Slf4j` 를 사용합니다.
- Controller, Repository 계층에서는 굳이 로깅하지 않습니다. 필요하다면 P6Spy, AoP 를 사용합니다.

## 레벨

**ERROR** — 즉시 대응이 필요한 장애 상황에서 사용합니다. 

외부 시스템 연동 실패, 데이터 정합성 깨짐, 예상하지 못한 예외 등이 해당됩니다. `@RestControllerAdvice`에서 처리되지 않는 예외가 여기에 속합니다.

```java
try {
    paymentClient.request(orderId);
} catch (PaymentException e) {
    log.error("[Payment] 결제 요청 실패 orderId={}", orderId, e);
    throw e;
}
```

**WARN** — 당장 장애는 아니지만 주의가 필요한 상황에서 사용합니다. 

비즈니스 예외(`CustomException`), 재시도 발생, deprecated API 호출 등이 해당됩니다.

```java
log.warn("[Order] 재고 부족으로 주문 실패 productId={}, requested={}, available={}",
    productId, requestedQty, availableQty);
```

<aside>
💡

**ERROR**, **WARN** 와 같은 예외 객체는 반드시 마지막 인자로 넣어 스택트레이스가 출력되도록 합니다. 

</aside>

**INFO** — 비즈니스 흐름의 주요 분기점에서 사용합니다. 

주문 생성, 결제 완료, 회원 가입 같은 상태 변경 이벤트가 해당됩니다. 단, 반복 호출되는 조회 API에서는 사용하지 않습니다.

```java
log.info("[Order] 주문 생성 완료 orderId={}, memberId={}, totalAmount={}",
    order.getId(), memberId, totalAmount);
```

**DEBUG** — 개발/디버깅 시 필요한 상세 정보에서 사용합니다. 

쿼리 파라미터, 중간 연산 결과, 외부 API 요청/응답 등이 해당됩니다. 운영 환경에서는 기본적으로 꺼둡니다.

```java
log.debug("[Order] 할인 계산 결과 originalPrice={}, discountRate={}, finalPrice={}",
    originalPrice, discountRate, finalPrice);
```
