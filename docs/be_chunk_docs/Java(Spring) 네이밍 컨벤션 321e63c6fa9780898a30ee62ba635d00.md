### 파일 & 디렉토리

- 파일 : 파스칼 케이스 (ex : `GlobalException.java` )
- 디렉토리 : 소문자, 복합어가 필요하다면 스네이크 케이스 (ex: `golf_course`)

### 클래스 & 인터페이스

- 클래스 : 파스칼 케이스 (ex : `SensorService`)
- 인터페이스 : 파스칼 케이스 (ex : `Runnable`)

### 변수 & 함수

- 필드 : 카멜케이스
- 메서드 : 카멜케이스
- 불리언값 : is, has, can 등의 접두사 사용

### **DTO**

- DTO는 레이어별로 분리를 지향합니다.
    - Controller, Service, Repository 에서 사용하는 DTO를 각각 만들어 서비스 레이어의 재사용성을 높이고 변경에 대한 사이드이펙트를 줄입니다.
    - 불필요한 분리라면 굳이 분리가 없어도 됩니다.
    - 레이어별로 아래와 같은 접미어를 붙입니다.
    - Controller : `Req`  `Res`
    - Service : `Command`
    - Repository : `Query`
    - ex) 유저를 찾는 API 에서
        - Controller DTO : `FindUserReq`
        - Service DTO : `FindUserCommand`
        - Repository DTO : `FindUserQuery`
- DTO 컨벤션은 `{동작}{도메인}{추가옵션}` 를 사용합니다.
    - ex)
        - `FindUserListReq`
        - `FindUserListRes`
            - `FindUserListItemRes`
        - `FindUserWithAccountReq`
        - `FindUserRes`
        - `UpdateUserRequest`
- DTO → 다른 DTO 또는 다른 엔티티로의 변환은 DTO에서 책임집니다.
    - from (다른 DTO 또는 엔티티로부터 생성) : static 메서드로 생성
    - to (현재 DTO를 다른 DTO 또는 엔티티로 변환) : 일반 메서드로 생성
    - 메서드 이름은 from, to 만 사용함으로써 자바의 **오버로딩**을 적극 활용합니다.
    - **변환 매퍼를 따로 빼지 않고 변환 주체가 책임집니다.**
    
    ```java
    public class FindLatestVersionRes {
    
      private Long id;
      private AppPlatform platform;
      private String version;
    
      public static FindLatestVersionRes from(AppVersion appVersion) {
        return FindLatestVersionRes.builder()
            .id(appVersion.getId())
            .platform(appVersion.getPlatform())
            .version(appVersion.getVersion())
            .build();
      }
      public AppVersion to() {
        return AppVersion
            .builder()
            .id(id)
            .version(version)
            .build();
      }
    }
    ```
