## Title

Git 협업 컨벤션 개요

---

## Rule

이 문서는 프로젝트에서 사용하는 **Git 협업 규칙의 전체 구조를 설명한다.**

Git 협업 규칙은 다음 네 가지 문서로 구성된다.

Git Commit Message Convention

Git Branch Strategy Convention

Git Pull Request Convention

각 문서는 Git 협업 과정의 서로 다른 단계에 대한 규칙을 정의한다.

Commit Message Convention은 **커밋 메시지 작성 규칙**을 정의한다.

Branch Strategy Convention은 **브랜치 생성 및 운영 전략**을 정의한다.

Pull Request Convention은 **코드 리뷰와 협업을 위한 PR 작성 규칙**을 정의한다.

---

## Rationale

Git 협업 과정은 일반적으로 다음 단계로 진행된다.

코드 변경

→ 커밋 생성

→ 브랜치 관리

→ Pull Request 생성

→ 코드 리뷰 및 병합

각 단계를 별도 문서로 관리하면 다음 효과가 있다.

- 협업 규칙 이해도 향상
- LLM multi-document reasoning 가능

---

## Exception

소규모 프로젝트에서는 일부 Git 규칙이 단순화될 수 있다.

---

## Override

팀의 협업 방식에 따라 브랜치 전략 또는 PR 프로세스는 변경될 수 있다.
