## Title

프론트엔드 변수 및 함수/컴포넌트 선언 컨벤션

## Rule

### 변수 선언

- 일반 변수는 `camelCase`를 사용한다.

### 상수 객체

- 상수 객체 내부 속성은 `camelCase`를 사용한다.

### 이벤트 핸들러

- 컴포넌트 내부 이벤트 핸들러는 `handle{CamelCase}` 형식을 사용한다.

### Props 핸들러

- 컴포넌트 props 핸들러는 `on{CamelCase}` 형식을 사용한다.

### 컴포넌트 선언

- React 컴포넌트 이름은 `PascalCase`를 사용한다.

### 페이지 컴포넌트

- 웹 페이지는 `{PascalCase}Page` 형식을 사용한다.

### 모바일 화면 컴포넌트

- React Native 화면은 `{PascalCase}Screen` 형식을 사용한다.

### API 함수

- API 호출 함수는 `{domainCamelCase}API` 형식을 사용한다.

예시:

- `handleSubmit`
- `onSubmit`
- `LoginPage`
- `LoginScreen`
- `getUserProfileAPI`

## Rationale

코드 이름은 역할과 동작을 명확하게 표현해야 한다.

## Exception

- 외부 API 필드명은 예외를 허용한다.

## Override

- 특정 프레임워크 패턴을 따르는 경우 override를 허용한다.
