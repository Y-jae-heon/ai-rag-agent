# Title

UI 구조 규칙

# Rule

UI 컴포넌트는 다음 위치 기준으로 배치한다.

| 위치 | 기준 |
| --- | --- |
| shared/ui | 범용 UI |
| entities/*/ui | 도메인 UI |
| features/*/ui | 사용자 행동 UI |
| widgets/*/ui | 페이지 블록 UI |

예

```
shared/ui
 ├ Button
 ├ Input
 ├ Dialog
```

```
entities/user/ui
 ├ UserAvatar
 └ UserBadge
```

```
features/user/create/ui
 ├ CreateUserForm
 └ CreateUserDialog
```

```
widgets/user-management/ui
 └ UserManagementPanel
```

Atomic Design 구조는 **폴더 구조로 사용하지 않는다.**

금지 구조

```
shared/ui/atoms
shared/ui/molecules
shared/ui/organisms
```

Atomic Design은 **컴포넌트 설계 철학으로만 사용한다.**

예

```
Button → atom
SearchForm → molecule
UserCard → organism
```

# Rationale

Atomic Design을 폴더 구조로 적용하면 다음 문제가 발생한다.

- UI 분류 기준 증가
- FSD slice 구조와 충돌
- 컴포넌트 위치 판단 어려움

따라서 UI 구조는 **FSD 기준 하나만 사용한다.**

# Exception

디자인 시스템 프로젝트에서는 Atomic 구조를 사용할 수 있다.

예

```
design-system
 ├ atoms
 ├ molecules
 └ organisms
```

# Override

UI 구조 변경은 디자인 시스템 관리 정책에 따른다.
