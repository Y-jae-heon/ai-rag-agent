## Title

프론트엔드 타입 및 인터페이스 선언 컨벤션

## Rule

### 타입 선언

- Type 이름은 `PascalCase`를 사용한다.

### 인터페이스 선언

- Interface 이름은 `PascalCase`를 사용한다.

### Enum 선언

- Enum 이름은 `PascalCase`를 사용한다.
- Enum 값은 `SCREAMING_SNAKE_CASE`를 사용한다.

### API 타입

- 서버 통신 타입은 역할 기반 suffix를 사용한다.

예시:

- `LoginRequest`
- `LoginResponse`
- `SessionDto`

## Rationale

타입 이름은 데이터 구조와 역할을 명확하게 표현해야 한다.

## Exception

- 자동 생성 타입은 예외를 허용한다.

## Override

- OpenAPI 또는 GraphQL 코드 생성 규칙을 따르는 경우 override를 허용한다.
